#1. Macbook
  #a. It is convenient to run Ollama from terminal instead of running it as a service.
If running as a service, then use commands like these to set env params: launchctl setenv OLLAMA_HOST "127.0.0.1:9000"
If running as a terminal command then the variables have to be specified as such: OLLAMA_HOST=0.0.0.0:11434 ollama serve 

  #b. For Chrome to access local service by machine name, Settings -> Privacy & Security -> Local Network, and enable both Google Chrome and Google Chrome Helper

#2. VScode + Ollama via Continue.dev
  - In VScode install extention Continue.dev
  - For config, config.yaml will open, edit it like this sample -
name: Local Config
version: 1.0.0
schema: v1
models:
  - name: qwen2.5-coder:7b
    provider: ollama
    model: qwen2.5-coder:7b
    apiBase: http://localhost:11434
    roles:
      - chat
      - edit
    capabilities: # Add if not auto-detected
      - tool_use
  - name: qwen3.5:4b
    provider: ollama
    model: qwen3.5:4b
    apiBase: http://localhost:11434
    roles:
      - autocomplete
[Ignore all search results that refer to config.json]

#3. Fine-tuning qwen3:8b
  - write changes in a file like: my-qwen3-8b.txt
  - ollama create my-qwen3-8b -f my-qwen3-8b.txt
  - ollama run my-qwen3-8b
  - example my-qwen3-8b.txt:
               FROM qwen3:8b
               SYSTEM """You are a highly concise assistant. 
               Give short, direct answers without unnecessary explanations or filler. 
               If a one-word or one-sentence answer suffices, use it."""

               PARAMETER num_predict 100
               PARAMETER presence_penalty 1.5
               PARAMETER temperature 0.7
               PARAMETER top_p 0.8
